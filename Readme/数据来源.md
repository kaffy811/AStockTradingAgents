# 数据来源

### 1. A 股主要获取了什么数据和信息？

针对 A 股，系统主要获取以下四个维度的数据：

- **基础资产信息**：公司名称、所属行业、上市市场、上市日期、实时/盘后快照价格、成交量与涨跌幅。
- **技术面行情数据（量价与K线）**：过去 365 天的历史日线量价数据（开盘价、收盘价、最高价、最低价、成交量）。并在底层计算滚动技术指标（如 MA5/10/20/60、MACD、RSI、布林带中/上/下轨）。
- **基本面财务数据**：关键财务比率及估值指标（如市盈率 PE、市净率 PB、PEG、净资产收益率 ROE、每股收益 EPS、资产负债率等），以及近期财报的关键利润/营收表现。
- **新闻与舆情数据**：来自于东方财富、CCTV 财经等渠道的近期相关新闻头条、新闻摘要、发布时间，以及新闻的情感极性（正面/负面/中性）。

---

### 2. 不同智能体场景下调用了什么数据及获取方式？

系统中各智能体（Agents）被赋予了不同的工具（Tools），它们获取数据并在流水线中协作，具体分工如下：

| 智能体角色 (Agent) | 调用的核心数据 | 获取方式与使用工具 |
| --- | --- | --- |
| **基本面分析师**<br>(Fundamentals Analyst) | **公司财报与估值指标**（PE、PB、PEG、ROE、当前股价及历史10天兜底价格）。用于判断股价是否被低估/高估，并给出合理的**目标价位区间**。 | 强制调用 `get_stock_fundamentals_unified` 工具。该工具底层调用 AkShare 或 Tushare 自动拉取最新的财务报表及估值指标快照，组装成结构化文本喂给 LLM。 |
| **技术面(市场)分析师**<br>(Market Analyst) | **历史与当前量价、技术指标**（均线排布、MACD金叉/死叉、RSI超买超卖、支撑/压力位、布林带带宽）。用于判断短期/中期价格趋势。 | 强制调用 `get_stock_market_data_unified` 工具。只传入当前日期，工具底层自动向前回溯提取 365 天的 A 股日线数据（利用 Pandas 和 Numpy 预计算上述技术指标并拼装），LLM 根据预计算指标进行逻辑解读。 |
| **新闻 / 情绪分析师**<br>(News & Social Media) | **近期个股新闻头条、摘要与情感倾向**。用于分析宏观经济环境、政策变化以及社交媒体对该公司的短线多空情绪。 | 调用 `get_unified_news_tool` 统一新闻工具。对于 A 股，底层自动降级调用东方财富（`stock_news_em`）获取近期新闻，并通过系统模块打上初步的情感标签，交由 LLM 生成深度情绪报告。 |
| **交易员**<br>(Trader) | **不直接请求外部 API 数据**。它调用前序三个分析师产出的《基本面报告》、《技术面报告》、《新闻/情绪报告》汇总文本。 | 从 LangGraph 的**全局状态字典 (State)** 中读取上游节点写入的报告，并从 ChromaDB 向量数据库提取**历史记忆 (Past Memory)**（避免重复犯错），从而生成一个初步的《投资建议计划》。 |
| **风控主管**<br>(Risk Manager) | **不直接请求外部 API 数据**。它审阅交易员的初步计划，并评估风险辩论组（激进、中立、保守三位分析师）的激辩记录。 | 从状态字典读取交易员计划、辩论历史以及检索匹配的历史失败教训。它纯靠 LLM 进行逻辑统筹，输出最终的一锤定音决策（买入/持有/卖出）并附带止损、止盈的风险提示。 |

**总结**：该系统的数据流设计非常精妙。**前置分析师（基本面、技术面、新闻）** 作为“数据收集与单一维度解读层”，强制使用封装好的 Python 数据工具访问真实金融 API 获取数据；**后置决策者（交易员、风控主管、研究总监）** 作为“逻辑决策层”，被剥夺了访问外部 API 的工具权限，专注于根据前置报告和向量数据库记忆，进行推理和统筹拍板。


### 一、 港股数据是如何获得的？

Agents（如 `Market Analyst`，`Fundamentals Analyst`）不会直接硬编码调用特定的 API，而是通过调用类似 `get_hk_stock_data_unified`、`get_stock_market_data_unified` 和 `get_stock_fundamentals_unified` 这样的工具函数。这些底层机制如下：

1. **主选数据源：AKShare (新浪财经 & 东方财富)**
   - 框架优先使用开源的 `AKShare` 库抓取港股数据。
   - 其中历史行情和实时报价调用了新浪财经相关的接口（如 `ak.stock_hk_daily`, `ak.stock_hk_spot`）。
   - 财务指标调用了东方财富的接口（如 `ak.stock_financial_hk_analysis_indicator_em`）。
   - 为了防止被 API 封禁，项目中特别针对 AKShare 设计了**线程锁（Threading Lock）**、**请求速率限制（Rate Limit）** 和 **内存/JSON文件缓存（Cache）** 机制。

2. **备用数据源：Yahoo Finance (`yfinance`)**
   - 当 `AKShare` 抓取失败或超时，系统会自动降级并尝试使用 `yfinance`（雅虎财经）获取对应的港股数据（通过 `yf.Ticker(symbol).history()` ）。

3. **内置映射与本地缓存**
   - 项目在 `ImprovedHKStockProvider` 中内置了一批知名港股（如腾讯、阿里、美团等）的代码到公司名称的映射字典，避免为了获取基础名字去浪费 API 请求次数。此外，所有成功获取的财务数据和基本信息都会被缓存在本地（如 `hk_stock_cache.json`）。

### 二、 分别获得了什么数据？

根据不同 Agent 的分析需求，底层接口向外提供的数据可以分为三大类：

#### 1. 市场行情与技术指标数据（主要供 `市场分析师 Market Analyst` 使用）
*从 `AKShare (新浪)` 或 `yfinance` 获取，并在本地经过 pandas 计算加工：*
- **基础行情**：日期 (Date)、开盘价 (Open)、最高价 (High)、最低价 (Low)、收盘价 (Close)、昨收价 (Pre-close)、涨跌额 (Change)、涨跌幅 (Pct_change)、成交量 (Volume)。
- **实时盘口 (Spot)**：当前最新价、当日最高/最低/成交量等。
- **技术指标 (Indicators)**：
  - 移动平均线（MA）：MA5、MA10、MA20、MA60。
  - MACD 指标：DIF、DEA、MACD 值。
  - RSI 指标：RSI(14)。
  - 布林带（BOLL）：上轨 (Upper)、中轨 (Mid)、下轨 (Lower)。

#### 2. 财务与基本面数据（主要供 `基本面分析师 Fundamentals Analyst` 使用）
*主要通过 `AKShare (东方财富)` 的 `stock_financial_hk_analysis_indicator_em` 接口获取：*
- **估值指标（动态计算）**：市盈率（PE）、市净率（PB）。
- **每股指标**：基本每股收益 (EPS_Basic)、滚动每股收益 (EPS_TTM)、每股净资产 (BPS)、每股经营现金流。
- **盈利能力**：净资产收益率 (ROE)、总资产收益率 (ROA)、净利率 (Net Profit Ratio)、毛利率 (Gross Profit Ratio)。
- **营收与成长性**：营业收入及同比增长率 (YoY)、归母净利润及同比增长率 (YoY)。
- **偿债能力**：资产负债率 (Debt Asset Ratio)、流动比率 (Current Ratio)。
- **基本属性**：报告期 (Report Date)、财年 (Fiscal Year)。

#### 3. 股票基础信息（全局通用）
- **基本资料**：股票代码、公司中文名称、所在市场（港股）、交易所（HKG）、计价货币（HKD）等。
