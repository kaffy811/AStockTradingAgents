# A股港股数据来源

### **一、A 股（中国市场）数据**

**1. 数据来源：**

- **AKShare**：开源免费的数据源，用于获取 A 股的基础信息、K 线数据、财务指标及新闻（如东方财富个股新闻）。
- **Tushare**：作为另一个重要的数据获取源。
- **BaoStock**：作为一个免费且稳定的历史数据来源，主要用作 A 股历史数据的兜底补充。

**2. 获取方式：**

- **统一接口调用**：通过内部封装的 `get_china_stock_data_unified` 等统一函数获取数据。
- **支持灵活切换**：通过 `switch_china_data_source(source)` 方法，用户或系统可以在 `tushare`、`akshare`、`baostock` 这几个数据源之间动态切换配置。

---

### **二、港股市场数据**

**1. 数据来源：** 由于此前港股存在因单一使用 Yahoo Finance 导致被限制（Rate Limit）的问题，该系统设计了数据源优先级模式（多数据源互为备用）：

- **AKShare（优先级最高）**：官方文档中提到“港股数据优先走 AKShare”，通过调用新浪财经或东方财富的接口来获取港股 K 线、基础信息、财务指标和港股新闻。
- **Yahoo Finance (yfinance)**：作为获取港股基础信息和 K 线数据的备选。
- **Finnhub**：作为第三方 API 补充，用于获取港股的基础信息、K 线以及新闻数据。

**2. 获取方式：**

- **多源降级重试策略**：统一调用 `get_hk_stock_data_unified` 接口，底层 `ForeignStockService` 以及统一数据源管理器会按数据库配置的优先级依次尝试。
    - 例如获取基础信息，会依次尝试 `_get_hk_info_from_akshare` -> `_get_hk_info_from_yfinance` -> `_get_hk_info_from_finnhub`，一旦某个触发限制或出错，则自动切换到下一个可用数据源以保证高可用性。
- **代码补齐兼容**：获取时如果港股代码格式不规范（如不是5位），底层工具会自动执行填充（如 `symbol.zfill(5)` 或加上 `.HK` 后缀）进行适配。
