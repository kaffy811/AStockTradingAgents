# 功能介绍

| 功能模块 | 实现方式 | 核心工具 / 技术栈 |
| --- | --- | --- |
| **多智能体协作网络 (Multi-Agent)** | 采用**图计算化 (Graph) 的编排模式**，将不同投研角色（基本面分析师、技术面分析师、情绪分析师、交易员、风控主管）定义为图中的节点 (Nodes)。通过状态机在节点之间传递全局状态（如：股票数据、买卖信号），通过条件边 (Edges) 实现多链路流转、智能路由与任务回退。 | **LangGraph**, Python, 大语言模型 (LLM) |
| **多大语言模型 (LLM) 聚合与路由** | 构建了统一的 `llm_clients` 抽象层和 Provider 标准化工厂。系统将外部模型（OpenAI, DeepSeek, Qwen等）的参数与环境密钥统一映射。当任务触发时，根据不同分析师的能力需求（智能模型选择）动态初始化模型通道，并统一收口 API 请求。 | **Python**, OpenAI SDK, 厂商 REST APIs, AiHubMix/OpenRouter 等聚合渠道 |
| **多市场金融数据源与容灾降级** | 封装 `ForeignStockService` 和统一的数据管理器接口。采用**多级异常捕获与降级链（Fallback）**机制。例如：获取港股行情时，首选 `AkShare`；若触发反爬或限流，系统捕获异常后自动切换降级至 `Yahoo Finance`，最后兜底至 `Finnhub`。 | **AkShare**, **Tushare**, **BaoStock**, **yfinance**, Finnhub, Pandas, Numpy |
| **企业级前后端分离架构** | **后端**采用现代异步框架提供 RESTful API，处理智能体调度与高并发请求；**前端**采用组件化、响应式的现代前端框架构建单页应用 (SPA)，实现流畅的无刷新交互与配置管理。 | 后端：**FastAPI**, Uvicorn<br>前端：**Vue 3**, **Vite**, **Element Plus**, TypeScript |
| **实时进度交互与长连接推送** | 由于智能体链式思考和数据获取耗时较长，系统在后端通过异步生成器捕获 LangGraph 的节点执行事件和 LLM 的流式输出。通过双通道建立服务端与客户端的持续连接，将实时分析进度、错误原因打回前端展示。 | **WebSocket**, **SSE (Server-Sent Events)**, FastAPI |
| **数据持久化与高并发缓存** | 采用**双数据库架构**：结构化/半结构化数据（如系统配置、模型参数、历史报告、用户权限）存入文档数据库持久化；高频的股票实时行情、API请求数据则存入内存数据库进行多级缓存，减少 API 速率限制并提高响应速度。 | **MongoDB** (持久化存储),<br>**Redis** (高速缓存 / 状态管理) |
| **技术指标与因子计算** | 在本地提取完金融基础行情和 K 线数据后，直接利用数据分析科学库进行滚动窗口 (Rolling) 计算，得出 MA 均线、MACD、RSI、布林带等量化指标，将计算结果结构化后喂给 LLM 进行逻辑解读。 | **Pandas**, Numpy |
| **专业投研报告导出** | 大模型完成所有链路分析并合成最终结果后，输出标准化的 Markdown 文本。系统后端内置文档转换器，将 Markdown 渲染并解析为多媒体文档流供前端下载。 | Markdown, 常见 Python 报表生成库 (如 python-docx / pdfkit) |
| **容器化与跨架构部署** | 编写 `Dockerfile` 对前后端环境进行隔离封装，并通过 `docker-compose` 编排服务与数据库的启动顺序。配置挂载卷 (Volumes) 保证数据不丢失，并结合 CI/CD 工具自动构建兼容 x86_64 和 ARM64 的镜像。 | **Docker**, **Docker Compose**, GitHub Actions |
