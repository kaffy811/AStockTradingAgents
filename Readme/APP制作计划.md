# APP制作计划

将 `TradingAgents-CN` 系统转化为移动端 APP，最核心的架构原则是：**“重后端，轻前端”**。

因为多智能体协作、大模型推理（LLM）、LangGraph 状态图流转和大量金融数据拉取都是计算密集型任务，这些必须保留在现有的 **FastAPI + 云端服务器**上运行。APP 端只作为一个“展示界面”和“交互终端”。

以下是为您梳理的 APP 化完整实现路径与技术方案：

### **第一步：选择移动端技术栈**

鉴于该项目的 Web 端是基于 Vue 3 构建的，您可以根据团队技术背景选择以下跨平台方案：

1. **UniApp (推荐)**：基于 Vue 语法。您可以最大程度地复用现有 Web 端的前端业务逻辑和状态管理，一套代码编译出 iOS、Android、微信小程序和 H5。
2. **Flutter**：性能最好，自带极其丰富的原生动画和 UI 组件，非常适合渲染复杂的金融图表（使用 `fl_chart` 或引入 Echarts）和流畅的报告展示。
3. **React Native**：生态成熟，如果是熟悉 React 的团队首选，对于集成 WebSocket 和 SSE（流式输出）有极好的社区支持。

### **第二步：后端 (FastAPI) 的适配改造**

现有的 FastAPI 后端已经具备了 RESTful API 和 WebSocket/SSE。为了适配 APP，需要做少量改造：

1. **移动端鉴权 (Auth)**：如果目前是 Session 方案，需改为标准的 **JWT (JSON Web Token)**，配合 Refresh Token 机制，因为 APP 端需要长期保持登录状态。
2. **网络重连机制**：移动端网络不稳定，当 APP 切入后台或断网时，SSE（Server-Sent Events）或 WebSocket 流容易断开。后端需增加“任务恢复/状态查询”接口，让 APP 重新打开时能拉取当前智能体的分析进度。
3. **消息推送 (Push Notification)**：分析一次股票通常需要几分钟。需集成极光推送、苹果 APNs 或 FCM。当分析流结束（达到 `END` 节点）时，后端给 APP 发送推送通知：“您对 腾讯控股 (0700.HK) 的基本面分析已完成，请查看报告”。

### **第三步：APP 核心模块与 UI 设计**

APP 的交互可以参考传统的炒股软件（如同花顺）结合 AI 对话软件（如 ChatGPT）。

- **首页 (自选与市场)**：展示用户关注的自选股列表。
- **智能体工作台 (核心功能)**：
    - **输入区**：输入股票代码或语音输入（“帮我分析一下贵州茅台”）。
    - **进度展示**：由于后端 LangGraph 是逐步执行的，APP 页面应设计一个**进度骨架屏或时间轴 (Timeline)**，通过 SSE 实时展示：“正在获取A股数据... -> 基本面分析师正在撰写财报解读... -> 风险主管正在评估...”。
- **报告详情页 (Markdown + 图表)**：
    - 展示 LLM 最终输出的 Markdown 格式报告。
    - 结合移动端图表库，将获取到的历史 K 线数据渲染为移动端可视化的交互图表。
- **我的 / 设置页**：
    - 复用现有的配置管理，允许用户在 APP 内填写自己的大模型 API Key（OpenAI、DeepSeek、阿里云等）。

### **第四步：移动端需要攻克的技术难点**

1. **流式输出 (SSE/Stream) 接收**：移动端原生发起 SSE 请求不如 Web 浏览器（EventSource）方便。
    - *方案*：在 Flutter 中可使用 `http` 库监听字节流，或在 React Native 中使用 `react-native-sse`，逐字渲染大模型的输出效果。
2. **Markdown 的原生渲染**：大模型输出的是包含复杂表格、加粗、甚至 LaTeX 公式的 Markdown 报告。
    - *方案*：必须引入支持移动端的 Markdown 渲染器（如 Flutter 的 `flutter_markdown`，或利用 WebView 嵌套渲染）。
3. **动态图表渲染**：
    - *方案*：股票 K 线图和技术指标在原生移动端开发成本较高，最简单的降级方案是在 APP 内嵌一个轻量级 WebView，运行基于 ECharts 或 TradingView 的轻量级 H5 图表。

### **总结实施路线图：**

1. **MVP 阶段 (最小可用产品)**：保持后端不变，用 UniApp 快速将现有的 Vue 页面打包成 H5 App (WebView 套壳)，打通账号登录和基本的图文报告查看。
2. **V1.0 阶段**：用 Flutter/React Native 重写核心的**智能体流式对话页面**和**报告渲染页**，优化移动端请求 SSE 的体验。
3. **V2.0 阶段**：增加原生图表、消息推送、后台运行机制、移动端本地 SQLite 缓存历史分析报告。
