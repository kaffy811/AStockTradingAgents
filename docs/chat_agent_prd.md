## 1. 产品定位

TradingAgents Chat Copilot 是基于现有股票研究工作台的对话式 AI 研究助手。它通过自然语言理解用户意图，调用行情、新闻、财务、报告、自选股、行业与对比工具，完成多步骤股票研究任务，并将结果沉淀到现有页面系统中。

**它不是：**
- 荐股机器人
- 自动交易系统
- 通用个人助手

**它是：**
- 金融垂直场景的 agentic research copilot
- 现有工作台功能的自然语言入口
- 多步骤研究流程的自动化编排器

**核心理念：** 从"用户自己找功能"变成"用户提出研究目标，Agent 调用工具完成任务"。

**架构定位（OpenClaw-inspired）：** Chat Copilot 不只是问答框，而是金融垂直场景的 Agent Orchestrator：理解研究意图 → 调用工具 → 记忆上下文 → 执行多步任务 → 结果落地到报告/自选/对比/行业页。完整架构演进见 [`docs/openclaw_inspired_roadmap.md`](openclaw_inspired_roadmap.md)。

## 3. 用户痛点

1. **不知道从哪里开始**：新用户面对首页 ComprehensiveAnalysisView，不知道该输入什么股票代码、选择哪种分析范围。
2. **不知道看哪些指标**：即便拿到报告，用户不理解 ROE、PE、MA20 等指标含义与关联。
3. **看完报告仍抓不住结论**：虽然 M51 已做结论先行，但用户仍可能希望"用更简单的话解释一下"。
4. **想追问报告，但报告是静态页面**：报告生成后是只读 Markdown，用户无法追问"为什么这样判断"。
5. **多步骤操作需多次点击**：把股票加入自选、生成报告、创建对比，需要在 3 个不同页面分别操作。
6. **研究流程分散**：异动分析 → 新闻拆解 → 同行对比 → 生成报告，需要用户自己串联步骤。

## 4. 核心价值

> 从"用户自己找功能"变成"用户提出研究目标，Agent 调用工具完成任务"。

**具体价值：**

| 场景 | 旧体验 | 新体验 |
|------|--------|--------|
| 生成报告 | 打开首页 → 输入代码 → 选范围 → 等待 | "帮我分析一下 688146 的技术面" |
| 加入自选 | 进入股票详情页 → 点击"加入自选" | "把中船特气加入我的自选股" |

| 解释报告 | 静态阅读，无法追问 | "这份报告里的均线多头排列是什么意思" |
| 行业研究 | 进入行业页 → 选行业 → 查看热股 | "最近电子行业有哪些热门股" |

## 5. 核心场景

### 场景 1：单股异动原因分析

**用户输入：**  
"中船特气今天为什么大涨"

**Agent 执行步骤：**  
1. `resolve_stock_tool`：解析"中船特气"→ CN/688146  
2. `get_quote_tool`：获取今日行情（涨幅、成交量）  
3. `get_latest_news_tool`：获取近 72 小时新闻  
4. `get_kline_summary_tool`：获取近期技术面简报

**调用工具：** resolve_stock / get_quote / get_latest_news / get_kline_summary  
**返回结果：** 涨幅 + 新闻关键事件（六氟化钨概念）+ 技术面简报  
**是否需要确认：** 否（只读）  
**落地页面：** `/stocks/CN/688146`（点击可进入详情页）

---

### 场景 2：单股风险优先分析

**用户输入：**  
"告诉我贵州茅台现在主要的风险点是什么"

**Agent 执行步骤：**  
1. `resolve_stock_tool`：解析"贵州茅台"→ CN/600519  
2. `get_fundamentals_tool`：获取财务指标（PE、PB、负债率）  
3. `get_latest_news_tool`：获取近期新闻  
4. `get_kline_summary_tool`：技术面支撑位判断  
综合 3 个维度输出风险摘要

**调用工具：** resolve_stock / get_fundamentals / get_latest_news / get_kline_summary  
**返回结果：** 结构化风险清单（估值风险、技术风险、新闻风险），附加免责声明  
**是否需要确认：** 否  
**落地页面：** `/stocks/CN/600519`

---

### 场景 3：生成综合报告

**用户输入：**  
"帮我生成一份 688146 的综合分析报告"

**Agent 执行步骤：**  
1. `resolve_stock_tool`：解析代码  
2. 确认："我将为 中船特气（CN/688146）生成 comprehensive 分析报告，是否确认？"  
3. 用户确认后：`create_analysis_run_tool`（scope=comprehensive）  
4. 等待 SSE 完成事件，返回报告摘要  
5. 提示"点击查看完整报告"

**调用工具：** resolve_stock / create_analysis_run  
**返回结果：** 综合结论卡片摘要 + 跳转链接  
**是否需要确认：** 是（long_running 操作）  
**落地页面：** `/history/{report_id}`

---

### 场景 4：解释历史报告

**用户输入：**  
"解释一下上次那份中船特气的报告，均线多头排列是什么意思"

**Agent 执行步骤：**  
1. `get_recent_reports_tool`：查找最近关于 CN/688146 的报告  
2. `get_report_detail_tool`：获取报告内容  
3. 从技术面章节提取均线相关段落  
4. 用通俗语言解释"均线多头排列"

**调用工具：** get_recent_reports / get_report_detail  
**返回结果：** 报告摘要 + 名词解释  
**是否需要确认：** 否  
**落地页面：** `/history/{report_id}`

---

### 场景 5：自选股巡检

**用户输入：**  
"帮我看看我的自选股今天怎么样"

**Agent 执行步骤：**  
1. `get_watchlist_tool`：获取用户自选股列表

3. 汇总：涨跌幅排行、异常波动提示

**调用工具：** get_watchlist / get_quote（批量）  
**返回结果：** 自选股今日行情汇总表格 + 异常波动提示  
**是否需要确认：** 否  
**落地页面：** `/watchlist`

### 场景 6：多股对比

**用户输入：**  
"帮我对比一下 600519 和 000858 这两只股票"

## 6. MVP 范围

### 包含（MVP）

| 功能 | 说明 |
|------|------|
| Chat 页面（`/chat`） | 消息列表 + 输入框 + 工具调用状态展示 |
| 单 session 对话 | 当前会话上下文记忆，不跨会话 |
| 只读工具调用 | 查询行情、新闻、基本面、行业、报告 |
| 生成报告工具 | create_analysis_run（需确认） |
| 加入自选工具 | add_to_watchlist（需确认） |
| 跳转对比页工具 | create_compare_selection（需确认） |
| 读取历史报告 | get_recent_reports + get_report_detail |
| 简单结构化记忆 | 当前 session 消息 + 最近股票 + 用户语言偏好 |
| 安全护栏 | 金融表达过滤、prompt injection 防护 |
| 审计日志 | 工具调用记录（日志级别） |

### 不包含（MVP 范围外）

| 功能 | 原因 |
|------|------|
| 真实交易、自动下单 | 核心禁止项，永久不做 |
| 目标价预测、买入/卖出建议 | 合规禁止项 |
| 未经用户确认的写操作 | 安全要求 |
| 长期向量记忆（Embedding DB） | 复杂度过高，MVP 暂缓 |
| 跨会话记忆持久化 | 需要新 DB 表，MVP 暂缓 |
| 多用户协作 / 共享报告 | 超出 MVP 范围 |
| 手机原生 App | PWA 已覆盖 |
| WebSocket 实时行情 | SSE 已满足研究场景 |
| 港股 Chat 覆盖 | 数据源不稳定，MVP 聚焦 A 股 |
| 多轮 Planner（超过 5 步） | C7 阶段再做 |

## 7. 成功指标

| 指标 | 标准 |
|------|------|
| 自然语言生成报告 | 用户能通过一句话触发报告生成，无需打开首页 |
| 解释历史报告 | 用户能追问历史报告内容，获得通俗解释 |
| 加入自选 | 一句话完成自选股添加，含确认流程 |
| 创建多股对比 | 一句话触发对比页跳转，URL 参数正确 |
| 工具轨迹可见 | 80% 以上回答附带工具调用轨迹（展示 Agent 思考过程） |
| 写操作确认率 | 100% 写操作包含用户确认步骤 |
| 合规输出 | 任何回答中不出现"买入"/"卖出"/"持有"/"目标价" |
| 落地页跳转 | 100% 结果卡片包含跳转到现有页面的链接 |
